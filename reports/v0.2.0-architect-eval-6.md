## 4. What v0.2.0 delivers

v0.2.0 is the "Living Organization" release — the static franchise of v0.1.0 becomes a continuously-churning org with a working minor-league pipeline, a transaction market, front-office turnover, and an organic news feed, with the LLM demoted from decision-maker to narrator.

- **Live minor-league pipeline (AB-10 — the four-iteration Critical, RESOLVED via Option C).** Injury IL-stints vacate the 25-man slot (`writeGame` + engine twin); recovery restores it; within-5 send-downs fire; `options_remaining` is seeded by service time at worldgen (veterans with 3+ years start option-less). The full state machine — call-up → send-down → DFA → waiver → claim — fires organically on every fresh season (Adversary organic probes: ~134–148 call_up, 21–29 send_down, 4–8 dfa, 3–5 waiver_claim per season across seeds 7/11/42), with zero manual DB seeding. Roster invariants hold (`over25Teams=0`).
- **Transaction market & waiver wire.** DFA'd players hit waivers; other orgs claim them. New `GET /api/waivers`, `waiverCount` on `/api/state`, per-player `GET /api/players/:id/transactions`.
- **Front-office turnover.** GM/manager firings with a job-security model and `manager_resigned` events; interim replacements with a mid-season stability floor; GM archetypes; owner death/personality flowing into front-office events and the news feed.
- **Live minors stats.** `GET /api/teams/:id/minors` now returns live (synthesized) minor-league stats rather than static rows; age/level-gated `dev_tick` rating changes for prospects.
- **Organic news feed.** All 15 news event types fire from real sim events (injuries, milestones, call-ups, firings, owner death, waiver claims, etc.); `GET /api/news` with lowercase `?type=` filtering (400 on non-allowlisted tokens, by design — CB-3).
- **LLM restructure — narrator only.** Every decision (firings, call-ups, draft picks, send-downs) is now **procedural**; the LLM produces flavor (headlines, ~20 sampled draft-pick reasonings) and never makes a decision. Zero LLM decision calls in the tick path; clean procedural fallback when no API key is present.
- **Shippable production artifact.** The built server applies all 8 migrations on a fresh DB, self-heals a migration-pending (carried) DB on boot, and launches via `node dist/server/server/index.js` (AB-19 fix). Security: `security:bundle-grep` key-leak gate clean; `scrubError` redacts `x-api-key` in error paths; all DB writes parameterized and league-scoped.
- **419 tests** passing (95% coverage thresholds), including the honest `iter5LiveOrgChain.test.ts` integration gate (asserts the chain fires with no manual seeding; fails against pre-fix commits) and the corrected `buildPackaging.test.ts` (asserts the runtime-read migration path).

## 5. PM handoff notes — for `app-spec.md` on merge

The PM should update `app-spec.md` (and reconcile the test spec) with the following when merging v0.2.0:

### New tables
- **`news_items`** — organic news feed; backs `GET /api/news`; carries event type, headline (LLM or procedural), and references to the originating entity.
- **`waiver_wire`** additions — DFA'd players placed on waivers; claim state machine; backs `GET /api/waivers` and `waiverCount`.
- (Migration 008 adds `players.injury_return_game` for the IL-stint return tracking.)

### New endpoints
- **`GET /api/waivers`** — current waiver-wire contents (200, array; empty `[]` when none).
- **`GET /api/news`** — news feed; supports `?type=<token>` filtering with **lowercase** canonical tokens; non-allowlisted tokens return **400** `{"error":"Invalid event type filter"}` (this is spec/security-correct — do NOT add case-insensitivity; uppercase strings like `INJURY`/`MILESTONE` are *badge labels*, not filter tokens).
- **`GET /api/players/:id/transactions`** — per-player transaction history.

### Modified endpoints
- **`GET /api/teams/:id/minors`** — now returns **live (synthesized) minor-league stats** rather than static rows (display-only synthesis; not a `dev_tick` rating change).
- **`GET /api/state`** — adds **`waiverCount`**.

### New fields on `teams`
- **`gm_archetype`**, **`interim_gm`** (flag), **`interim_manager`** (flag) — front-office turnover model.

### LLM restructure (architecture note)
- **All decisions are now procedural; the LLM is a narrator only** (headlines + ~20 sampled draft-pick reasonings). No LLM decision calls in the tick path. Document the procedural-first architecture and the procedural fallback behavior when `ANTHROPIC_API_KEY` is absent. **LLM headline / draft-reasoning quality should be verified in a keyed env before public release** (the test env runs keyless, so headlines are empty strings and normal-pick reasoning is null — expected, not a defect).

### Production server launch (ops section)
- **`node dist/server/server/index.js`** (i.e., `npm start`) — note the nested `server/server/` path is a consequence of `tsconfig.server.json`'s `include: ["server","shared"]` preserving the common source root. Migrations must reside at `dist/server/server/migrations/` (the loader reads `join(__dirname, 'migrations')` from the compiled `db.js`); the build's `copy-migrations` step now writes there. (Optional future polish, non-blocking: a `rimraf dist` prebuild step to avoid a harmless stale `dist/server/migrations/` dir lingering across builds.)

### Spec corrections noted by testers (doc-only, carried from Eval-5 — reconcile, no code change)
- **`/api/league`** is not implemented — league info is in `/api/state` (`leagueId` present); the app works without `/api/league`.
- **`/api/transactions`** has **no `type` filter** — it returns the 50 most-recent transactions by `created_at DESC`. Use `/api/news?type=` for filtered event views and `/api/players/:id/transactions` for per-player history. (`dev_tick` dominating the unfiltered list is correct.)
- **v0.2.0 data-testid reference:** nav testids are **`nav-{section}`**; standings table is **`league-standings-table`**; minors tab is **`team-minors-tab`**. `roster-player-{id}` and **Owner-in-team-detail-panel** are NOT v0.2.0 deliverables — clarify whether they're in scope for a later version.
- **Trade-floor (`is_on_25man`) verification** requires a fresh-world sim through game ~30–40 (restored-DB substrate is unreliable for this check).
- **Interim-manager re-fire** behavior should be re-checked on a fresh world (the Eval-5 black-box observation was on a carried Season-11 DB tracing to pre-Iter-5 data; current-source unit tests for firing cadence pass). The spec's "interims cannot be fired mid-season (stability floor)" is the intended contract.
