## Critical Issue Discovered: Migration 008 Not Auto-Applied

**The server emits `[db] No migrations directory found` on startup.** Migration `008_injury_return.sql` (which adds `injury_return_game INTEGER` to players) exists in `dist/server/migrations/` but the server cannot locate it at runtime. This caused **every sim tick to crash** with `[engine] Tick error: no such column: injury_return_game` beginning at game 296, stalling the sim completely.

**Resolution:** Manually applied the migration via sqlite3:
```sql
ALTER TABLE players ADD COLUMN injury_return_game INTEGER;
INSERT INTO schema_versions (version, applied_at) VALUES (8, ...);
```
After restart, sim advanced normally. **This is a P0 regression — the sim is unrunnable on a fresh restored DB without manual intervention.**

- **Call-up rate extremely low** — 7 total in 22 seasons vs Adversary claim of 134-148/season

- **/api/transactions type filter non-functional** — always returns dev_tick records regardless of type= param
- **Uppercase news type filter** — INJURY/MILESTONE still return 400
- **Draft pick reasoning strings missing** — Only "Pool exhausted" messages, not actual selection reasoning
- **Call-up events not written to news_items** — call_up transactions occur but never appear in news feed

6. **LLM circuit breaker open** — By end of test session, `llmStatus.circuitBreakerOpen: true` (auth failures triggered circuit breaker). Will prevent news headline generation if key is later provided.
