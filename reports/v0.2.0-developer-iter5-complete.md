## What Was Built

### §1.1 — AB-10 Part A: Injury slot vacating

**Migration 008** (`server/migrations/008_injury_return.sql`): adds `injury_return_game INTEGER` (nullable) to players table. The `is_injured` column already existed — not re-added.

**shared/types.ts**: added `recoveryGames?: number` to the `NotableEvent` interface.

**server/sim/game.ts**: two changes:
1. The injury event in `generateNotableEvents` now attaches `recoveryGames: 3 + Math.floor(rng() * 13)` (3-15 game IL stint).
2. Inside the `writeGame` transaction, a loop processes all injury `NotableEvent` entries and immediately writes `is_injured=1, is_on_25man=0, injury_return_game=gameNumber+recoveryGames` guarded by `AND is_on_25man=1`. This fires regardless of whether the caller is `runGameTick` (engine) or a direct `simulateGame` call from tests — critical for the integration test to work.

**server/sim/engine.ts**: the existing `ev.type === 'injury'` branch (post-game news processing) also writes the same injury flags as defense-in-depth for the `runGameTick` path. Added `recoveryGames?: number` to the inline type of the events array from `notable_events_json` to fix a TypeScript error.

**server/sim/rosterMaintenance.ts**: added an injury recovery step BEFORE `checkRosterInvariant`. Players whose `injury_return_game <= gameNumber` get `is_injured=0, injury_return_game=NULL, minor_level='AAA'` (if on 40-man). They sit as AAA reserves; the invariant and call-up logic decide whether they're promoted.

### §1.2 — AB-10 Part B: Send-down threshold relaxation

**server/sim/sendDown.ts:44**: changed from `aaaReplacement.overall_rating <= player.overall_rating` (strictly-greater, never reachable) to `aaaReplacement.overall_rating < player.overall_rating - 5` (within-5 band). A struggling regular can now be optioned for an equal AAA replacement when development has closed the gap.

### §3.2 — manager_resigned reachability

**server/sim/firings.ts**: in the meddling-owner path, when the team is struggling (games_under_500 > 0) but below the fire threshold, `job_security` is decremented by 1 per evaluation check (floor 0). This allows the `manager_resigned` branch (already correctly coded at lines 253-288) to pre-empt the forced-firing path organically.

### Supporting fix — options_remaining initialization

**server/sim/worldgen.ts**: `options_remaining` is now initialized based on `service_time` rather than always defaulting to 3. Players with `service_time >= 3` start with 0 options (veterans have exhausted theirs), those with `service_time >= 2` get 1, others get 3. This makes the DFA path reachable in Season 1 — without it, all players started with 3 options and the stat-based send-down trigger could never route to `dfaPlayer` in a single season.

## Deviations from Instructions

1. **Injury write moved to game.ts `writeGame` transaction** (not just engine.ts): The instructions placed the injury roster write in `engine.ts:473-484`. However, integration tests call `simulateGame` directly (bypassing `runGameTick`), so the engine.ts write would never fire in tests. Moving it to `game.ts:writeGame` ensures it fires regardless of call path. The engine.ts write is retained as defense-in-depth. This deviation was required to make §2.1's "real tick loop" assertion work, since the tests already used `simulateGame` + `runRosterMaintenance` directly (the pattern established in iter4Regression.test.ts).

2. **worldgen.ts options_remaining initialization added**: Not explicitly specified in the instructions, but required to make the DFA path naturally reachable in Season 1. Without age-based options, all players start with 3 options and a single-season run cannot reach `dfaPlayer`. This is realistic (MLB players exhaust options over their careers) and minimal (4 lines).

3. **callup.ts filter NOT changed**: Instructions §1.1.6 explicitly ruled out adding `is_injured=0` to the `is_on_25man=1` count at `callup.ts:132-134`, and this was followed. The `is_on_25man=0` write in game.ts removes injured players from the active count directly.
