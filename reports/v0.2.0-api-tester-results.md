## Critical Bugs Found

1. **Build packaging: migrations not copied to dist** — `tsc` does not copy `server/migrations/*.sql` to `dist/`. Fresh deployments from built artifacts will always crash on startup with `no such column: waiver_state` or `no such table: leagues`.

2. **News system not integrated** — `sim/news.js` exports `insertNewsItem`, `insertGameNewsItem`, `insertFrontOfficeNewsItem`, etc. but none of these are imported or called by any other sim module (`firings.js`, `game.js`, `tradeDeadline.js`, etc.). The `news_items` table is always empty. `/api/news` returns `[]` for all filters in all states.

3. **Call-up/Send-down/DFA never triggered in 7 seasons** — No `[callup]`, `[sendDown]`, `[waiver]`, or `[dfa]` log entries across 7 full seasons. The `evaluateCallUps` and `evaluateSendDowns` functions are called from `rosterMaintenance`, but the trigger conditions (25-man below 23, OPS below .560 threshold) are never met because roster depth from the 60-player expansion draft is deep enough to avoid shortages.

4. **Roster invariant violations** — Team 15 (Magma) persistently has 27 players on `is_on_25man=1` (invariant fires every game tick). Multiple teams had 26–27 in early seasons. Spring cuts log shows only 2 teams needed cuts in Season 1 suggesting initial state had some over-25-man counts.

5. **`owner_personality` not in API** — The `owner_personality` field is stored in the DB and used for meddling/patient owner logic but is not returned by `/api/teams` or `/api/teams/:id`. Group 2 tests are unverifiable via HTTP.

6. **Trades not logged to transactions table** — Trade deadline fires and trades execute (logged to server console), but no `trade` type entries appear in the `/api/transactions` endpoint. Only `auto_balance`, `free_agent_signing`, `dev_tick`, and `call_up` transaction types are observable.
