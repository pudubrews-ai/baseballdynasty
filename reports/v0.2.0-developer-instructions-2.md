### §1.3 — [FIX C-3] Make call-ups / send-downs / DFA actually reachable

The hook is wired (`engine.ts:431` → `rosterMaintenance.ts:86-90`) but the triggers never fire across 500+ games. Fix reachability:

1. **Lower the roster-shortage trigger to bite, and add a routine churn path.** `callup.ts:122` `active25Man < 23` rarely happens with no injury depletion. Per spec ("MLB roster drops below 23 active (injuries)"), there IS no injury system depleting the active count. Two required changes:
   - **(a)** Implement at least a minimal injury/IL effect so the 25-man can drop below 23 — OR (simpler, acceptable) reinterpret Trigger 1 as `is_on_25man < 25` (restore-to-25), since spring cuts and send-downs reduce the count. The spec's intent is "roster has a hole → fill it." Choose the restore-to-25 approach: if `is_on_25man < 25`, call up the best available at the most-needed position. This guarantees call-ups fire whenever a send-down or DFA opened a slot.
   - **(b)** Ensure send-downs CAN fire (see §2.7 — recent windows): once `recent_*` slides, a genuinely slumping MLB player will trip the OPS/ERA trigger and be sent down, opening a slot that (a) then refills. This is the churn loop the spec describes.
2. **The DFA path is gated behind send-down-no-options and call-up-needs-40-man-slot.** Both become reachable once (1) restores roster churn and (2) §2.7 lets send-downs fire. Verify with the §5 integration test that at least one DFA and one waiver resolution occur in a multi-season sim.
3. **Service time:** call-ups set `first_mlb_call_up_game` (`callup.ts:88`); once players actually move between MLB and AAA, `service_time_days` will accrue for the right players (API Tester saw all-zero because nobody ever reached MLB via call-up).

**Why:** The live minor-league system is the second headline feature and is currently 100% dormant.
