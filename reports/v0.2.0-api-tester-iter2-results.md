## Critical Bugs Found

### BUG-1: Waiver Wire Never Populates (Group 4 — Critical)
- `waiverCount` always 0 across 4 full seasons of play
- No DFA events in news feed whatsoever
- `/api/waivers` returns `[]` throughout all regular season play
- The entire waiver wire system does not appear to be firing

### BUG-2: No Mid-Season Call-Ups or Send-Downs (Group 5 — Critical)
- Zero `call_up` events in news across 4 seasons
- Zero mid-season `send_down` events (only at game_number=0 for spring training)
- Players stay on MLB roster regardless of performance

### BUG-3: GET /api/news?type=FRONT+OFFICE Returns 400 (Group 11 — Medium)
- Spec requires HTTP 200 for `?type=FRONT+OFFICE`
- Actual: HTTP 400 `{"error":"Invalid event type filter"}`
- Valid equivalent is `?type=frontoffice` (no space, lowercase)

### BUG-4: News Pagination (offset param) Not Functional (new endpoint)
- `?offset=N` parameter is ignored — always returns newest 50 events
- Cannot paginate through historical news

### BUG-5: front_office_events details_json Always Null (Groups 2, 8)
- All manager_fired, gm_fired, owner_sold_team events have `details_json: null`
- Spec requires logging departing and incoming person names
- Cannot verify who was fired/hired

### BUG-6: interim_manager Flag Inconsistency (Group 8 — Medium)
- Teams show `manager_name: "Interim Manager"` but `interim_manager: 0`
- Flag should be set when interim manager is installed

### BUG-7: Spring Training Release Type Is send_down Not release (Group 3)
- Spec expects `transaction_type = "release"` for spring training cuts
- Actual: `transaction_type = "send_down"`, players go to minors (not released to FA)
- Low-overall/D-potential players are not being released, they're being sent to minors
