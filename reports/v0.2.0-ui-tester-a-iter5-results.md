## 7. Outstanding Issues for Dev Team

### High Priority
1. **Spec/impl testid mismatch â€” nav tabs:** All main nav items use `nav-{section}` (e.g., `nav-teams`). Spec and tests assume `{section}-tab`. Either update spec to `nav-*` or add alias `data-testid` attributes.
2. **`standings-table` testid missing:** Actual testid is `league-standings-table`. Spec says `standings-table`. Needs alignment.
3. **`roster-player-{id}` testids absent:** Group 3 spec requires per-player testids on the roster tab. Currently the roster renders as plain text inside `team-detail-panel` without individual testids.

### Medium Priority
4. **Owner text not in team detail panel:** The spec (Group 2, Group 8) requires Owner name and personality visible in team detail. Currently only GM is shown. Either "Owner" text needs to be added to the team panel, or the spec's UI expectation needs clarification.
5. **`draft-tab` testid:** In the nav bar the element has testid `nav-draft`, not `draft-tab`. Inconsistent with spec.

### Low Priority / Observation
6. **game_result `headline_text` field:** Contains score string (e.g., "Ironport Jaguars 2, Coldbrook Foundry 4"). The spec says "score only â€” no LLM headline." This is correctly implemented â€” the `headline_text` holds the score, not an LLM narrative. Test logic that flags this as a failure is wrong; this is a PASS.
7. **`news-filter-transactions` count:** Showed 50 results (same as "all"). This may indicate the transactions filter is not filtering correctly, or all 50 events are considered transaction-type. Worker B should investigate.
