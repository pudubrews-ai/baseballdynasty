## New Endpoints (from Iteration 3 spec)

| Endpoint | Spec Expects | Actual | Result |
|---|---|---|---|
| GET /api/news?type=ROSTER | 200 | 400 "Invalid event type filter" | FAIL |
| GET /api/news?type=INJURY | 200 | 400 "Invalid event type filter" | FAIL |
| GET /api/news?type=MILESTONE | 200 | 400 "Invalid event type filter" | FAIL |
| GET /api/news?type=roster (lowercase) | — | 200, 56 items (send_down events) | INFO |
| GET /api/news?type=injuries (lowercase) | — | 200, 22 items | INFO |
| GET /api/news?type=milestones (lowercase) | — | 200, 2 items | INFO |
| GET /api/teams/:id/minors | live stats fields | 200 with season_stats nested | PARTIAL |

**Finding:** The news filter is case-sensitive. The spec calls for `ROSTER`, `INJURY`, `MILESTONE` (uppercase) to return 200, but the server only accepts lowercase (`roster`, `injuries`, `milestones`). Note: lowercase `roster` returns all `send_down` events (not just generic roster moves), `injuries` returns injury events, and `milestones` returns milestone events.

## Summary of Bugs Found

### Critical
1. **DFA never fires** — 162 games, 0 DFA events, waiver wire always empty (waiverCount: 0). Players being sent down without triggering DFA when they have no options.
2. **Call-ups never fire** — 0 call_up events or transactions in 162 games. Teams with players injured/in minors are not calling up replacements.
3. **rosterMaintenance invariant fires every tick** — `team null has 717 on 25-man` logged on every single sim advance. Mass data integrity issue with null-team players.

### High
4. **`interim_manager` flag not set** — Multiple teams have `manager_name = "Interim Manager"` but `interim_manager = 0`. Flag integrity broken.
5. **In-season firing stalled** — Season 11 at game 162 with teams at 4-12 (meddling owner) — no firings triggered. Firing threshold may be too high or evaluation not running.
6. **`owner_died` event never occurs** — Zero owner_died events in 11 seasons of the live league.

### Medium
7. **`/api/players/99999999/transactions` returns 200 `[]`** instead of 404 for nonexistent player.
8. **News uppercase filters return 400** — `type=ROSTER`, `type=INJURY`, `type=MILESTONE` all fail; should accept case-insensitively per spec.
9. **`details_json` null on owner events** — Departing/incoming owner names not logged.
10. **Trade rate below target** — ~2 trades per season observed; spec requires ≥ 3 per season.

### Low
11. **News default limit is 50** — Without explicit `limit=200+` parameter, trade events and older frontoffice events are hidden. A caller following spec (just `GET /api/news?type=TRANSACTION`) gets 400; correct is `type=transactions&limit=200`.
12. **Minors live stats nested** — Stats in `season_stats` sub-object rather than flat fields (`games`, `ab`, `avg`, `ip`, `era` are null).

## Valid News Filter Types (discovered)
- `frontoffice` — manager/gm/owner events
- `transactions` — signings, releases, trades, non-tenders
- `injuries` — injury events
- `milestones` — milestone events
- `roster` — send_down events
- `all` — all event types

Uppercase variants (`ROSTER`, `INJURY`, `MILESTONE`, `TRANSACTION`) all return 400.
