The producer is wired; it has nothing to consume.

**Totals: 2 Critical, 0 High, 2 Medium, 2 Low.** (Net improvement over Iter-2's 2C/2H/3M/2L, but the two headline-feature Criticals persist with new root causes.)

## CRITICAL FINDINGS

### AB-10 (Critical) — The live call-up/send-down/DFA/waiver chain is STILL inert at runtime; the AAA pipeline is essentially empty (3–7 AAA players across 20 teams, 0 with higher-rated same-position replacements), so every stat-based trigger fires into a vacuum

**Root cause:** The AB-01 ordering fix is correct (verified at `rosterMaintenance.ts:125-138` — evaluate first, reset after). But the call-up/send-down triggers all require an **AAA player at the same position** as the MLB player:
- Send-down Trigger 1 (`sendDown.ts:36-44`): needs `minor_level='AAA' AND position=? AND overall_rating > player.overall_rating`.
- Call-up Triggers 2/3/4 (`callup.ts:201-276`): all require `minor_level='AAA'` at matching position.

The minor-league system never holds AAA depth. Worldgen assigns minor leaguers a level, and the expansion draft stocks farms, but the result is overwhelmingly level `A`, with almost nothing at AAA/AA. There is also **no promotion path** from A→AA→AAA: `prospectDev.ts` (eligibility `minor_level IN ('AA','A','Rookie')`, `:43`) only ticks ratings up; it never changes `minor_level`. So AAA never fills over seasons either.

**File:line:** `worldgen.ts` farm assignment + `sendDown.ts:36-44` / `callup.ts:201-276` (same-position AAA requirement) + `prospectDev.ts:43` (no level promotion).

**Evidence (Adversary in-engine probes through the real loop):**
- Minor-level distribution league-wide (post expansion draft + spring cuts): seed 7 `{AAA:3, AA:0, A:97, Rookie:0}`; seed 11 `{AAA:5, AA:0, A:95}`; seed 42 `{AAA:5, AA:0, A:95}`; seed 99 `{AAA:7, AA:0, A:93}`. Team 1 (seed 7) has **zero** AAA players.
- 25-man position players with a higher-rated same-position AAA replacement: **0** on all four seeds.
- Full-season organic transaction counts: seed 7 `send_down:3 (all spring-cut, game 0), call_up:0, dfa:0, waiver_claim:0`; seed 11 `send_down:5 (spring), call_up:0, dfa:0, waiver_claim:0`. **`maxRecentAbSeen=20–24`** — confirming the window does accumulate past the 20-AB threshold, so the ordering fix genuinely works; the chain dies on the missing AAA replacement, not on the window.

**Why the green suite missed it:** `callupSenddownReachability.test.ts` (rewritten) drives the real `runRosterMaintenance` loop (good, proves ordering), but in `beforeAll` it **manufactures** the prerequisites the real sim never produces — it manually sets `recent_ab=25` (lines 76-83) AND **manually creates an AAA replacement at the right position rated +10** (lines 49-66) by re-leveling an `A` player or boosting an existing one. So the test proves "given a populated AAA and a low-OPS player, the loop fires a send-down" — true, but the real sim has neither the AAA depth nor (organically) reaches the trigger.

**Impact:** Fails Test Group 4 (call-ups) and Group 5 (send-downs/DFA/waivers) wholesale at runtime — identical user-visible outcome to the Iter-1 and Iter-2 Criticals. ROSTER-badge in-season news (call-ups, in-season send-downs, DFA), TRANSACTION-badge waiver-claim news, and `/api/waivers` content are all empty in normal play.

**Resolution required:** Stock the farm system with real AAA/AA depth (e.g., during worldgen/expansion-draft, distribute minor leaguers across AAA/AA/A by age/rating instead of dumping ~95% at `A`), AND/OR add a level-promotion path in `prospectDev`/offseason so high-A prospects graduate to AA/AAA over seasons. Then add an integration test that runs a **full season through `runGameTick` with no manual AAA seeding** and asserts in-season `call_up`/`send_down`/`dfa`/`waiver_claim` counts each ≥1.

### AB-11 (Critical) — INJURY events are mathematically impossible; MILESTONE events are effectively impossible in normal play. The producer is wired but has nothing to consume, so `news-filter-injuries` and `news-filter-milestones` are permanently empty

**Root cause (two structural dead paths):**

**(a) INJURY — generator range vs. trigger threshold mismatch.** `worldgen.ts:329` generates `injury_prone = randInt(rng, 3, 6)` — **maximum 6**. The in-game injury NotableEvent (`game.ts:761`) fires only when `player.injury_prone >= 7 && rng() < 0.05`. No generated player ever has `injury_prone >= 7`, so the condition is **unsatisfiable**. The injury-truncation path (`game.ts:830-848`) is dead for the same reason.

**(b) MILESTONE — thresholds unreachable from a cold start.** `checkCareerMilestones` (`game.ts:794-819`) fires on crossing 100/200 career HR, 2000 career hits, or 1000 career K. Every player starts at `career_hr=0, career_hits=0, career_k=0` (verified: max career_hr/hits/k = 0 after worldgen). Crossing 100 HR / 2000 hits / 1000 K takes many full seasons; my full-season probe crossed **none**.

**File:line:** `worldgen.ts:329` (injury_prone ≤ 6) vs `game.ts:761` (needs ≥ 7); `game.ts:794-819` (milestone thresholds) vs cold-start career 0.

**Evidence (Adversary probes):**
- `injury_prone` distribution league-wide (seed 7): `{3:185, 4:205, 5:202, 6:208}` — **`injury_prone>=7` count = 0**. Injury-eligible position players on 25-man with `injury_prone>=7`: **0**.
- `max career: hr=0 hits=0 k=0` after worldgen.
- Full-season organic probes (seeds 7 & 11): `injury NotableEvents = 0, milestone NotableEvents = 0`; news badges `INJURY:0, MILESTONE:0`.

**Why the green suite missed it:** `injuryMilestoneNews.test.ts` `beforeAll` force-sets `injury_prone = 10` on ALL players (line 34 — a value worldgen can never produce) and `career_hr = 99, career_hits = 1999` on ALL players (line 38 — impossible at world start), AND if still zero, **synthetically injects** an injury and a milestone news row with a pre-filled headline (lines 127-158). So the test validates the *wiring* under impossible inputs and a fallback; it cannot fail and proves nothing about real-sim reachability.

**Impact:** 2 of the 15 mandated news event types (spec §"Event Types That Generate News Items", lines 244-245) are unmet at runtime. The News tab's `news-filter-injuries` and `news-filter-milestones` always return empty. This is the same headline-feature completeness failure flagged in Iter-2 AB-02a; the wiring fix did not address why the events don't occur.

**Resolution required:** Either (a) raise the worldgen `injury_prone` range to include ≥7 (e.g., `randInt(rng, 3, 9)` or a weighted tail) so the existing `game.ts:761` injury path can fire, and/or lower the injury threshold to match the generated range; and (b) seed nonzero career stats at worldgen for veteran players (age-scaled `career_hr`/`career_hits`/`career_k`) so some players sit near milestone thresholds and cross them in normal play — or add lower in-season milestone tiers (e.g., 10/20/30 HR season, 100-hit season) that are reachable within a single season. Then add a no-force integration test asserting ≥1 INJURY and ≥1 MILESTONE news row over a multi-season real sim.

## MEDIUM FINDINGS

### AB-12 (Medium) — `manager_resigned` news event type has no producer

`manager_resigned` is in `NewsEventType`/`BADGE_MAP` (`news.ts:31,49`) and `proceduralHeadline` (`news.ts:278`), and the spec lists it (line 241), but grep finds no code path that resigns a manager or inserts a `manager_resigned` news item. Of the 15 spec event types this is a third that never appears at runtime (alongside INJURY/MILESTONE per AB-11). Lower severity than AB-11 because the spec firing chain (§"In-Season Firing Logic") describes firings, not resignations, so it is arguably an unimplemented optional event rather than a broken wired one. **Resolution:** either add a manager-resignation path (e.g., interim or veteran manager resigns under a meddling owner) or document `manager_resigned` as deferred/out-of-scope so the 15-type table is reconciled.

### AB-13 (Medium) — `forceMinimumTrades` last-resort tiers can move a starting/everyday MLB player as a "prospect," producing unrealistic trades to satisfy the floor

`findBuyerProspect` (`tradeDeadline.ts:265-294`) Tier 3 pulls a 40-man reserve and Tier 4 pulls *the lowest-rated 25-man bench player* (excluding only SP/CL/SS/C/CF). When the AAA pipeline is empty (the normal case per AB-10), the floor is met by shipping active 25-man roster players as the "package," which can produce baseball-nonsensical trades (a contender trading away an everyday LF to "acquire" a veteran it already has depth at). The floor is now guaranteed (good, AB-02b), but the realism cost is real and user-visible in the News feed. **Resolution:** prefer prospect-less/cash-equivalent packages over moving 25-man regulars, or restrict Tier 4 to clearly-surplus bench players; better still, fix AB-10 so AAA prospects exist and the natural `findTradePackage` path (AAA/AA only) carries most trades.

`callup.ts:209-220` comment says "The actual send-down will be handled by evaluateSendDowns," but `evaluateSendDowns` runs *before* `evaluateCallUps` in the same tick (`rosterMaintenance.ts:125-126`), so the displaced player is only trimmed by `checkRosterInvariant` (later in the same maintenance call) or the next tick. Latent today because AB-10 keeps this path from firing at all. **Resolution:** within the call-up, option the displaced starter in the same transaction, or document reliance on `checkRosterInvariant`.

### AB-15 (Low) — `injuryMilestoneNews.test.ts` is a tautology (impossible inputs + synthetic fallback), giving false confidence
Carried as a test-quality finding, not a product defect: the §5.3 test cannot fail regardless of product behavior (see AB-11 "why the green suite missed it"). It should sim realistically and assert organic events, or be explicitly labeled a wiring-only test with the reachability gap tracked separately. **Resolution:** tie the test to the AB-11 fix — assert organic INJURY/MILESTONE rows with no `UPDATE players SET injury_prone=10` / `career_hr=99` and no synthetic injection.

## VERDICT: NOT READY

Iteration 3 is a real, well-targeted iteration: it correctly fixed the reset-before-eval ordering inversion (AB-01), wired the INJURY/MILESTONE producers (AB-02a), made the ≥3-trades floor genuinely seed-independent (AB-02b — verified 3 trades on seed 7 and 4 on the previously-dry seed 11 through the real engine path), and fixed the root `.gitignore` so a clean checkout boots (AB-05). Four Iter-2 Mediums/Lows and the architect's additional Mediums also landed, and the suite is a clean 401/0. **But the two headline-feature Criticals survive with new root causes that the code-level fixes did not reach.** The live minor-league system still produces **zero** in-season roster moves at runtime — not because of the (now-fixed) window ordering, but because the AAA pipeline is essentially empty (3–7 AAA players across 20 teams, 0 with a higher-rated same-position replacement on any seed) and there is no A→AA→AAA promotion path, so every call-up/send-down trigger fires into a vacuum (AB-10; organic probes seeds 7 & 11 → 0 call-ups, 0 DFA, 0 waiver claims, 0 in-season send-downs). And INJURY news is **mathematically impossible** because worldgen caps `injury_prone` at 6 while the trigger needs ≥7, while MILESTONE news is effectively impossible from a cold career-0 start — so two of the fifteen mandated news event types remain permanently empty (AB-11; organic probes → 0 injuries, 0 milestones, badge counts INJURY:0/MILESTONE:0). Both green tests that "cover" these (`callupSenddownReachability`, `injuryMilestoneNews`) pass only by manufacturing prerequisites the real sim never produces (manual AAA seeding; `injury_prone=10` / `career_hr=99`; a synthetic-injection fallback).

**Required before READY:** (1) AB-10 — stock the farm system with real AAA/AA depth at worldgen/expansion-draft and/or add a level-promotion path, then add a no-manual-seeding full-season `runGameTick` test asserting in-season call-up/send-down/DFA/waiver counts ≥1 each; (2) AB-11 — raise the worldgen `injury_prone` range (or lower the `game.ts:761` threshold) so injuries can occur, and seed age-scaled career stats and/or add reachable in-season milestone tiers, then add a no-force multi-season test asserting ≥1 INJURY and ≥1 MILESTONE news row. AB-12/AB-13 (Medium) should land for completeness/realism; AB-14/AB-15 are low.
