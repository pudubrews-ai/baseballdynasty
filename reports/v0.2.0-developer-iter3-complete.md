## §1 Critical Fixes

### §1.1 — AB-01: rosterMaintenance.ts ordering fix

**File:** `baseball-dynasty/server/sim/rosterMaintenance.ts` (lines ~120-140)

The `recent_*` reset was running BEFORE `evaluateSendDowns()` / `evaluateCallUps()`, making the entire call-up/send-down/DFA/waiver chain structurally unreachable. Fixed by moving the reset AFTER both evaluators within the `if (callUpDue)` block:

```
1. evaluateSendDowns(team, ...) — reads accumulated recent_ab >= 20 etc.
2. evaluateCallUps(team, ...)   — reads recent_starts, recent_era etc.
3. THEN reset recent_* to 0     — cleared for the next 5-game window
```

### §1.2a — AB-02: INJURY + MILESTONE news producers wired

**File:** `baseball-dynasty/server/sim/engine.ts` (runGameTick)

Extended the `gameRow` SELECT to include `notable_events_json` and `id`. After inserting the game news item, the function now iterates over each `NotableEvent` and calls:
- `insertMilestoneNewsItem(...)` for `type === 'milestone'`
- `insertNewsItem({ eventType: 'injury', ... })` for `type === 'injury'`

`team_id` is resolved via `SELECT team_id FROM players WHERE id = ?` (structured columns, injection-safe per CISO F23). Both event types are in `BADGE_MAP` and handled by `proceduralHeadline` fallback.

Also extended `insertMilestoneNewsItem` signature in `news.ts` to accept `sourceTable` and `sourceId` params.

### §1.2b — AB-02: forceMinimumTrades non-determinism fixed

**File:** `baseball-dynasty/server/sim/tradeDeadline.ts`

Three changes:

1. **Progressive prospect search** (`findBuyerProspect` helper): Tier 1 = AAA/AA, Tier 2 = A/Rookie, Tier 3 = 40-man reserves, Tier 4 = lowest bench player. No more `continue` on missing AAA/AA.

2. **Exhaust buyer pairings**: Inner buyer loop tries each buyer in order; breaks only after a trade executes, not when one buyer lacks prospects.

3. **Distinct-trade count**: Added `countDistinctLeagueTrades()` which counts `news_items.event_type='trade'` (one row per `executeTrade` call vs. 1+N prospect rows in transactions). `forceMinimumTrades` now uses this count for the floor check.
