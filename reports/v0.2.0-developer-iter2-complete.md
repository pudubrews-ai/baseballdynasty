## §1 — What Was Fixed In This Iteration

### H-2: Spring cuts "exactly 25" enforcement

**Root cause:** `runSpringCutsForTeam` took an early return when `current25Man <= TARGET_25MAN` and called `repairPositionMinimums` → `fillTo25`. When repair added 2 players (SP + CL), the count went 25 → 27 with no trim loop in the early-return path.

**Fix:** Restructured `runSpringCutsForTeam` into 4 explicit steps that always run regardless of initial count:
1. Cut excess players (only if > 25)
2. Repair position minimums (always)
3. Trim back to 25 if repair pushed over (always — finds non-minimum-protected players)
4. Fill to 25 if short (always)

The trim step (Step 3) scans all 25-man players ordered by `overall_rating ASC` and finds the first whose removal doesn't violate `POSITION_MINIMUMS`. This prevents minimum-protected positions from being double-counted as eligible to cut.

**Verification:** All 11 spring cuts tests pass. Every team ends at exactly 25 on `is_on_25man=1`.
